ReplayVOD ships as a Docker Compose stack today. Native Windows `.exe` is supported by the installer once release archives are published; until then, use the Docker Desktop path on Windows.

## Requirements

- A Twitch developer application (for `TWITCH_CLIENT_ID` and `TWITCH_SECRET`).
- Your numeric Twitch user ID (for `OWNER_TWITCH_ID`).
- Docker (Docker Desktop on Windows).
- `ffmpeg` is bundled in the Docker image. Native Windows installs need `ffmpeg` on `PATH` unless the release archive includes it.

## Installer

<Tabs syncKey="platform">
  <TabItem label="macOS / Linux">

  ```bash
  curl -fsSL https://replayvod.com/install.sh | sh
  ```

  The installer:

  <Steps>

  1. Creates `~/replayvod/`.
  2. Writes `server/.env` from the example and generates `HMAC_SECRET` plus `SESSION_SECRET` locally.
  3. Prompts for `TWITCH_CLIENT_ID`, `TWITCH_SECRET`, and `OWNER_TWITCH_ID` if a terminal is available. The secret prompt is hidden.
  4. Starts ReplayVOD with `docker compose up -d --build`.

  </Steps>

  **Overrides**

  Pick a different directory or database profile:

  ```bash
  curl -fsSL https://replayvod.com/install.sh | \
    REPLAYVOD_DIR=/srv/replayvod REPLAYVOD_PROFILE=postgres sh
  ```

  Configure without starting containers:

  ```bash
  curl -fsSL https://replayvod.com/install.sh | REPLAYVOD_NO_START=1 sh
  ```

  </TabItem>
  <TabItem label="Windows (Docker)">

  ```powershell
  $env:REPLAYVOD_INSTALL_MODE = "docker"
  irm https://replayvod.com/install.ps1 | iex
  ```

  The installer:

  <Steps>

  1. Creates `%LOCALAPPDATA%\ReplayVOD\`.
  2. Writes `server/.env` from the example and generates `HMAC_SECRET` plus `SESSION_SECRET` locally.
  3. Prompts for `TWITCH_CLIENT_ID`, `TWITCH_SECRET`, and `OWNER_TWITCH_ID` if a terminal is available. The secret prompt is hidden.
  4. Starts ReplayVOD with `docker compose up -d --build`.

  </Steps>

  **Overrides**

  Use a different database profile:

  ```powershell
  $env:REPLAYVOD_INSTALL_MODE = "docker"
  $env:REPLAYVOD_PROFILE = "postgres"
  ```

## Manual Docker

If you prefer to clone and run yourself:

```bash
git clone https://github.com/befabri/replayvod.git
cd replayvod
cp server/.env.example server/.env
$EDITOR server/.env
docker compose --env-file server/.env --profile sqlite up -d --build
```

Use `--profile postgres` to run against Postgres instead.

## Build from source

Requires Go 1.26+, Node 22+, [Task](https://taskfile.dev/), and `ffmpeg` on `PATH`.

```bash
git clone https://github.com/befabri/replayvod.git
cd replayvod
task setup    # go mod download + npm install
task dev      # server on :8080, dashboard on :3000
task build    # production builds
```

## Required environment

After install, fill these in `server/.env`:

```text
TWITCH_CLIENT_ID=...
TWITCH_SECRET=...
OWNER_TWITCH_ID=...
HMAC_SECRET=...      # generated by installers
SESSION_SECRET=...   # generated by installers
```

For a public deployment also set:

```text
PUBLIC_BASE_URL=https://your-domain.example
```

The Compose file derives `CALLBACK_URL`, `WEBHOOK_CALLBACK_URL`, and `FRONTEND_URL` from `PUBLIC_BASE_URL`.
