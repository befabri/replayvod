ReplayVOD has two configuration files.

<CardGrid>
  <Card title="server/.env">
    Credentials, database selection, public URLs, local paths, Connect relay URLs. Treat as secret.
  </Card>
  <Card title="server/config.toml">
    Operational behaviour: download policy, scheduler intervals, logging, storage backend, health endpoint, title-tracking mode.
  </Card>

<FileTree>

- replayvod/
  - server/
    - .env             secrets, paths, network
    - .env.example     committed template — every supported var is listed and commented here
    - config.toml      tuning
  - docker-compose.yml
  - data/

</FileTree>

## Precedence

`server/.env` always wins. The Go process reads the env file first, then the TOML file. If a setting is exposed in both (rare), the env value takes precedence so containerised deployments can override TOML without rebuilding the image.

The Docker Compose file reads `server/.env` directly. A few `PUBLIC_*` variables are Compose-only — they shape what gets passed into the container without changing the stored values.

## Required values

```text
TWITCH_CLIENT_ID=...
TWITCH_SECRET=...
HMAC_SECRET=...
SESSION_SECRET=...
OWNER_TWITCH_ID=...
```

The installer generates `HMAC_SECRET` and `SESSION_SECRET`. For manual installs, run `openssl rand -hex 32` for each.

## Public URLs

For Compose deployments, set `PUBLIC_BASE_URL` and let Compose derive everything:

```text
PUBLIC_BASE_URL=https://replayvod.example
```

For separate services or manual runtime, set each URL directly:

```text
CALLBACK_URL=https://replayvod.example/api/v1/auth/twitch/callback
WEBHOOK_CALLBACK_URL=https://replayvod.example/api/v1/webhook/callback
FRONTEND_URL=https://replayvod.example
```

<Aside type="caution" title="Don't lose server/.env">
  `server/.env` contains the secrets used to encrypt sessions and verify webhook HMACs. Losing it invalidates sessions and can break EventSub verification.
</Aside>
